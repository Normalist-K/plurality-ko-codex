# 4-4 Property and Contract QA

## 요약 판정

전체적으로 장 전체가 빠짐없이 번역되어 있으며, 원문의 논지 전개와 문단 순서는 잘 보존되어 있다. Markdown heading, 구분선, 목록, 링크 URL, 각주 참조/정의는 원문과 대응하며, 이미지 경로, code fence, figure 구조는 원문에 없으므로 누락 이슈도 없다.

다만 법률적 재산권 분해(`usus/abusus/fructus`), 통계 수치 임계값, 절 제목의 범위 등에서 독자가 원문의 개념을 다르게 이해할 수 있는 지점이 있어 P1 수정이 권장된다. 그 밖에는 전문용어 일관성과 문장 밀도를 다듬으면 스터디용 가독성이 더 좋아진다.

## P0 구조/누락

해당 없음. 원문 장의 본문, 소제목, 목록, 링크, 각주가 번역본에 모두 대응한다.

## P1 의미 오역

- [P1] 번역 71 / 원문 `rise above single digits` / 현재 번역 `두 자릿수를 넘어서면`은 "두 자릿수 이상을 넘어설 때"로 읽혀 원문의 "한 자릿수를 넘어 두 자릿수에 이르면"과 임계값이 달라진다 / `한 자릿수를 넘어서면` 또는 `두 자릿수로 올라서면`으로 수정.
- [P1] 번역 99 / 원문 `ability to abuse or transfer the data to others (abusus)` / 현재 번역 `남용하거나 타인에게 이전할 능력`은 라틴어 재산권 속성 `abusus`를 악의적 남용으로 좁혀 읽게 한다 / `데이터를 변경·처분하거나 타인에게 이전할 능력(abusus)`처럼 처분권 의미를 살린다.
- [P1] 번역 141 / 원문 `### ⿻ real property` / 현재 번역 `### ⿻ 부동산`은 뒤이어 다루는 전자기 스펙트럼·인터넷 네임스페이스·재산 체계 전반의 논의를 토지/건물 재산으로 좁혀 보이게 한다 / 문맥상 `### ⿻ 현실의 재산` 또는 `### ⿻ 실물 재산`을 검토.

## P2 용어/문체

- [P2] 번역 46 / 원문 `The Intergalactic Computer Network` / 현재 번역 소제목이 영어로 남아 있어 앞뒤의 한국어 소제목 체계와 다소 어긋난다 / `은하간 컴퓨터 네트워크`처럼 번역하되, 고유 문헌명은 본문 링크에서 원제를 유지.
- [P2] 번역 61 / 원문 `a tighter accounting of the underutilization of digital assets` / 현재 번역 `저활용을 더 촘촘하게 회계`는 직역투가 강하다 / `디지털 자산의 저활용 실태를 더 정밀하게 파악` 등으로 조정.
- [P2] 번역 79, 85, 113 / 원문 `computation`, `computational`, `compute` / 현재 번역이 `연산`, `컴퓨팅`, `컴퓨팅 단위`로 섞여 있다 / 장 내 용어표 기준에 맞춰 `연산 자산`, `컴퓨팅 자원`, `연산 단위` 등 일관된 패턴을 정한다.
- [P2] 번역 79 / 원문 `proprietary clouds` / 현재 번역 `독점 클라우드`는 monopoly처럼 읽힐 수 있다 / `독점형 클라우드`, `폐쇄형 클라우드`, `사유 클라우드` 중 프로젝트 용어에 맞춰 통일.
- [P2] 번역 115 / 원문 `what should ... Party B ... learn about Party A's data?` / 현재 번역 `무엇을 알아야 하는가`는 의무처럼 읽힌다 / 데이터 공개 범위의 질문이므로 `무엇을 알 수 있어야 하는가` 또는 `무엇을 알게 되어야 하는가`로 완화.
- [P2] 번역 117 / 원문 `confidential computers` / 현재 번역 `기밀 컴퓨터`는 업계 용어로 다소 낯설다 / `기밀 컴퓨팅 장치` 또는 `기밀 컴퓨팅 환경`으로 다듬기.
- [P2] 번역 135 / 원문 `mediators of individual data (MIDs)` / 현재 번역 `개인 데이터 중개자`는 상업적 브로커 뉘앙스가 강하다 / `개인 데이터 매개자(MIDs)` 또는 `개인 데이터 중재자(MIDs)` 검토.

## P3 취향/윤문

- [P3] 번역 36 / 원문 `beautifully drawn Atlas of AI` / 현재 번역 `아름답게 그려낸`은 시각적 그림책처럼 읽힐 수 있다 / `섬세하게 그려낸` 또는 `탁월하게 묘사한` 정도로 윤문.
- [P3] 번역 103 / 원문 `mutually expected norms` / 현재 번역 `상호적으로 기대되는 규범`은 어색하다 / `상호 기대된 규범` 또는 `서로 기대하는 규범`으로 간결화.
- [P3] 번역 139 / 원문 `encoded in common sense and legal practice` / 현재 번역 `상식과 법적 실천 속에 깊이 부호화`는 직역감이 있다 / `상식과 법적 관행 속에 깊이 자리 잡다`로 자연스럽게 조정.
- [P3] 번역 152 / 원문 `a reasonable first mental model`, `to a first blush` / 현재 번역 `첫 번째 심성 모델`, `첫눈에`는 의미는 통하지만 학술 산문에서 어색하다 / `첫 번째 직관적 모델`, `거칠게 보면` 등으로 다듬기.

## 권장 수정안

- 우선 P1 세 항목을 먼저 반영한다. 특히 `single digits`와 `abusus`는 스터디 독자의 수치·개념 이해에 직접 영향을 준다.
- 절 제목 `⿻ real property`는 프로젝트 전체에서 `real property`를 법률 용어 `부동산`으로 고정했는지 확인한 뒤 결정한다. 이 장의 문맥만 보면 `현실의 재산` 쪽이 더 안전하다.
- 이후 `computation/storage/data`, `proprietary`, `confidential computing`, `MIDs` 같은 반복 용어를 장 단위로 통일한다.
- 마지막으로 긴 문장 일부를 한국어 호흡에 맞게 나누면 스터디용 가독성이 크게 좋아진다. 링크 텍스트와 각주 ID는 현재처럼 보존한다.

## 보류/주의

- `real property`는 법률 영어에서 `부동산`으로 번역될 수 있으므로, 프로젝트 용어집이나 인접 장의 번역 방침이 있다면 그 기준을 우선한다.
- `생성형 기반 모델(GFMs)`은 의미상 문제는 없으나, 다른 장에서 `생성 기반 모델`, `파운데이션 모델` 등으로 통일되어 있다면 맞춰야 한다.
- 각주 설명 일부는 한국어로 번역되어 있고 일부는 영어 원문으로 유지되어 있다. 구조 문제는 아니지만, v0.2 전체 편집 단계에서 각주 번역 범위를 일괄 결정하는 것이 좋다.
