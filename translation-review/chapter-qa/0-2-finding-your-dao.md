# 0-2-finding-your-dao QA

## 요약 판정

원문 25행/번역 25행으로 문단, 목록, 강조 표시, 원문 URL, Markdown 링크가 모두 대응한다. 원문과 번역 모두 H1 1개만 있으며, 이미지 경로, figure 구조, code fence는 없다. P0급 구조 누락이나 P1급 의미 오역은 발견하지 못했다.

번역은 전체적으로 충실하고 읽을 수 있는 수준이다. 다만 첫 독자 안내 장이라는 성격상 목록 문장의 직역투, `technical/technological`, `impact`, GitHub 협업 용어, `credit` 처리 등은 책 전체 용어와 문체에 맞춰 조금 더 자연스럽게 다듬는 편이 좋다.

## P0 구조/누락

- 해당 없음. Markdown heading, 굵게 표시된 핵심어, URL, Markdown 링크 URL은 원문과 대응한다. 이미지, figure, code fence는 원문과 번역 모두 없다.

## P1 의미 오역

- 해당 없음.

## P2 용어/문체

- [P2] 번역 7행/`topical, political or **current affairs** interest`/현재 `주제별, 정치적 또는 **시사** 관심`은 `topical`과 `current affairs`의 결이 어색하게 겹치고 한국어 명사구가 뻣뻣함/`주제 중심, 정치, 또는 **시사·현안**에 주된 관심이 있는 독자`처럼 관심사의 범주가 자연스럽게 드러나도록 조정.
- [P2] 번역 11행/`more **technical**, technological or engineering focus`/현재 `더 **기술적**, 테크놀로지 또는 공학 중심의 초점`은 `technical/technological`의 중복 뉘앙스를 한국어로 풀지 못하고 `초점이 있는 독자`가 직역투임/`더 **기술적**이거나, 테크놀로지·공학 쪽에 초점을 둔 독자`처럼 자연화.
- [P2] 번역 15행/`real-world **impact** in specific social sectors`/현재 `특정 사회 부문에서 현실 세계의 **영향**`은 `impact`가 사회적 성과·변화의 의미보다 막연한 영향으로 읽힘/`특정 사회 부문에서의 현실적 **효과**` 또는 책 전체 용어가 허용하면 `사회적 **임팩트**`로 조정.
- [P2] 번역 23행/`issue`, `pull request`/현재 `"issue"`, `"pull request"`를 영어 그대로 두어 GitHub 협업 용어의 한국어 독해가 조금 끊김/책 전체 방침에 맞춰 `이슈(issue)`, `풀 리퀘스트(pull request)` 또는 `풀 리퀘스트(PR)`처럼 병기.
- [P2] 번역 23행/`All contributions are credited`/현재 `모든 기여는 크레딧에 기록되며`는 영어 `credit`을 옮긴 느낌이 강하고 한국어 문서 문체에서 부자연스러움/`모든 기여는 명시되며`, `모든 기여자는 기록되며`, `모든 기여에는 크레딧이 부여되며` 중 하나로 정리.

## P3 취향/윤문

- [P3] 번역 3행/`those who hold a physical copy`/현재 `물리적 책을 든 독자들`은 의미는 통하지만 직역감이 있음/`인쇄본을 읽는 독자`, `종이책을 읽는 독자`처럼 자연화.
- [P3] 번역 7행/`begin at the beginning of the book, with the preface and read straight through`/현재 `서문과 함께 책의 처음에서 시작해 곧장 읽어 나가라`는 `with the preface` 연결이 어색함/`서문부터 책의 처음에서 시작해 끝까지 읽어 나가라`처럼 재배열.
- [P3] 번역 19행/`who find value wherever they begin`/현재 `시작한 곳에서 가치를 발견한 대부분의 독자`는 뜻은 맞지만 문장 흐름이 무겁고 조건절이 덜 자연스러움/`어디서 시작했든 그 지점에서 가치를 발견한 독자라면 대체로...`처럼 다듬기.
- [P3] 번역 21행/`out of date already`/현재 `이미 최신이 아니며 최신 버전은`은 `최신` 반복으로 다소 딱딱함/`이미 구판일 가능성이 높으며, 최신 버전은...`처럼 윤문.
- [P3] 번역 23행/`at https://github.com/pluralitybook/plurality`/현재 `장소는 다음과 같다:`는 영어식 안내문 느낌이 강함/`... 커뮤니티가 검토하게 할 수 있다. 저장소는 https://github.com/pluralitybook/plurality 이다.`처럼 문장 안에 자연스럽게 연결.
- [P3] 번역 25행/`You are the nobody.`/현재 앞에서는 `여러분`을 쓰다가 마지막에 `당신`으로 바뀌어 호칭 일관성이 흔들릴 수 있음/의도적으로 직접성을 살릴지 확인하고, 일관성을 우선하면 `그 "아무도"가 바로 여러분이다`로 조정.

## 권장 수정안

- 7행 목록 문장은 `주제 중심, 정치, 또는 **시사·현안**에 주된 관심이 있는 독자는 서문부터 시작해 책을 처음부터 끝까지 읽어 나가라.` 정도로 정리하면 안내문 리듬이 좋아진다.
- 11행은 `더 **기술적**이거나 테크놀로지·공학 쪽에 초점을 둔 독자는 4부에서 시작하는 것을 고려하라.`를 권장한다.
- 15행은 책 전체 용어 기준에 따라 `현실적 **효과**` 또는 `사회적 **임팩트**` 중 하나로 고정한다.
- 23행은 `여러분은 언제든 텍스트에 대한 우려나 문제를 이슈(issue)로 제출해 커뮤니티가 우선순위를 정하게 하거나, 수정안을 풀 리퀘스트(pull request)로 제출해 커뮤니티가 검토하게 할 수 있다.`처럼 GitHub 용어를 병기하면 스터디 독자가 따라가기 쉽다.
- 25행 마지막 문장은 호칭 정책을 통일한다. 이 장 전체의 `여러분` 톤을 유지하려면 `그 "아무도"가 바로 여러분이다.`가 가장 안정적이다.

## 보류/주의

- `Dao`를 제목에서 `도(道)`로 옮긴 선택은 적절해 보이나, 책 전체에서 `Dao/道/도` 표기 원칙이 따로 있다면 그 기준을 우선해야 한다.
- `the material has no copyright`의 `저작권이 없다`는 원문에 충실하지만, 프로젝트 라이선스/퍼블릭도메인 표기 정책이 있다면 `저작권 제한이 없다`, `저작권이 적용되지 않는다` 등으로 법적 뉘앙스를 재검토할 수 있다.
- 원문 URL `https://www.plurality.net/`, GitHub 저장소 URL, `[All contributions are credited]` 링크 URL은 번역에서 보존되어 있다.
