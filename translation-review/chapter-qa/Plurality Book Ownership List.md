# Plurality Book Ownership List QA

## 요약 판정

조건부 통과. 원문과 번역 모두 7개 열, 8개 데이터 행의 Markdown 표 구조를 유지하고 있으며, 이름, 이메일, 빈 셀, 주요 역할 목록도 누락 없이 보존되어 있다. 원문에는 heading, 링크 URL, 이미지 경로, code fence, figure 구조가 없으므로 해당 보존 이슈는 없다. 다만 ownership list 성격상 `Ownership` 계열 용어와 일부 역할명이 약간 과번역 또는 어색하게 읽히는 부분이 있어 P2/P3 수준의 다듬기를 권장한다.

## P0 구조/누락

- 해당 없음. 표의 열/행 수, 인명, 연락처, 빈 셀, `Japanese` 항목 대응이 모두 보존되어 있다.

## P1 의미 오역

- 해당 없음. 역할과 담당 영역의 핵심 의미는 모두 전달되어 있다.

## P2 용어/문체

- [P2] 1행 헤더 / `Ownership` / 현재 번역 `소유/담당 영역`은 `Ownership List` 문맥에서 개인별 책임·오너십 영역을 뜻하는데, `소유`가 물권적 의미로 읽혀 다소 어색함 / `담당 영역` 또는 원문 감각을 살린 `오너십/담당 영역`으로 조정 권장.
- [P2] Andrea Bonaceto 행 / `Website/Book Visuals` / 현재 번역 `웹사이트/도서 비주얼`은 의미는 통하지만 `도서 비주얼`이 한국어 목록 문체에서 어색함 / `웹사이트/책 시각 자료` 또는 `웹사이트/책 비주얼`로 조정 권장.

## P3 취향/윤문

- [P3] 1행 헤더 / `First Name`, `Last Name` / 현재 번역 `이름`, `성`은 사용 가능하나, 한국어 독자에게 `이름`이 전체 이름처럼도 읽힐 수 있음 / 표 스키마 보존이 중요하면 `이름(First Name)`, `성(Last Name)`처럼 병기하는 방안 검토.
- [P3] 1행 헤더 / `Responsibilities` / 현재 번역 `책임`은 직역투가 약간 있음 / 실제 업무 항목을 넣는 열이라면 `담당 업무`가 더 자연스러움.

## 권장 수정안

- `소유/담당 영역` -> `담당 영역` 또는 `오너십/담당 영역`
- `웹사이트/도서 비주얼` -> `웹사이트/책 시각 자료`
- 선택: `책임` -> `담당 업무`
- 선택: `이름`/`성` -> `이름(First Name)`/`성(Last Name)`

## 보류/주의

- 원문 마지막 데이터 행은 파일 끝 줄바꿈이 없어 `wc -l` 기준으로 번역본과 줄 수가 다르게 보일 수 있으나, 논리 행 수는 둘 다 10행으로 동일하다.
- 원문 Jason Entenmann 행 끝에는 불필요한 trailing spaces가 있으나, 번역본에서 이를 보존하지 않은 것은 Markdown 표 렌더링상 문제로 보지 않는다.
- 이 파일은 소유권 목록이므로 인명, 이메일, 역할 구분 슬래시(`/`)는 현재처럼 원형 보존을 우선하는 편이 좋다.
